# Redirecting to the Hosted Payment Page

With the requestId obtained from the API, the final step is to redirect the user to the SnapPay Hosted
Payment Page. The URL is constructed by appending the requestId to the SnapPay Hosted Page URL.



import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<Tabs
    defaultValue="java"
    values={[
        {label: 'Java', value: 'java'},
        {label: 'C#', value: 'c#'},
    ]}>
<TabItem value="java">

```java showLineNumbers"
String HostedPageUrl = "https://snappaydirect-
cert.fiserv.com / interop / HostedPaymentPage / ProcessRequest ? reqNo = " +
response.getRequestid();
log.info("Snappay Hosted page: {}", HostedPageUrl);
HostedPageUrl =
    $ "https://snappaydirect-
cert.fiserv.com / interop / HostedPaymentPage / ProcessRequest ? reqNo = {
    requestID
}
";
```

</TabItem>

<TabItem value="c#">

```csharp showLineNumbers"
HostedPageUrl =
  $"https://snappaydirect-
cert.fiserv.com/interop/HostedPaymentPage/ProcessRequest?reqNo={requestID}";
```

</TabItem>
</Tabs>

- **Redirection**: In an actual implementation, you would likely use Spring’s redirect functionality to
send the user to this URL, completing the payment flow.
